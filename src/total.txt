🌍 Nhóm 5: Mở cửa đón khách (Chạy CUỐI CÙNG)
Apply file: ingress-online-boutique.yaml

Lý do: Sau khi check toàn bộ 12 con Pod trên đều báo trạng thái 1/1 Running, bro mới thiết lập Ingress để nối tên miền (như online-boutique.hannover.local) trỏ vào trong cụm K8s.
Ingress Controller của bro đang "hứng" traffic ở cổng 30080 trên các Node vật lý (111, 112, 113), chứ không phải cổng 80. Cổng 80 mà bro thấy trong lệnh đó chỉ là cổng nội bộ (ClusterIP) của nó thôi.

🛠️ Sửa lại file online-boutique.conf cho chuẩn:
Bro mở lại file và sửa cái Port trong phần upstream thành 30080 nhé:

Nginx
# 1. Định nghĩa cụm K8s của bro
upstream boutique_cluster {
   // server [IP_ADDRESS]; 
   // server [IP_ADDRESS]; 
   // server [IP_ADDRESS]; 
}

server {
    listen 80;

    # 2. Tên miền để trình duyệt nhận diện
    server_name online-boutique.kihpn.local;

    location / {
        proxy_pass http://boutique_cluster;
        proxy_redirect off;

        # 3. Đóng gói đầy đủ Header gửi cho Ingress
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
🚀 Tại sao phải dùng 30080?
NodePort: K8s đang mở một "lỗ hổng" ở cổng 30080 trên toàn bộ các máy chủ vật lý của bro để đón khách từ bên ngoài vào.

Bản chất: Khi con Nginx .115 gửi yêu cầu đến 192.168.157.111:30080, máy chủ .111 sẽ hiểu ngay là: "À, yêu cầu này dành cho Ingress Controller, để tôi đẩy nó vào trong cụm".
